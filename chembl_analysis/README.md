# ChEMBL Leakage Analysis

This project investigates potential data leakage or unexpected cross-reactivity in the ChEMBL database by identifying documents where the same compounds are tested against different protein targets with high correlation in activity but low sequence similarity.

## Files

- `find_matches.py`: The core script that queries a ChEMBL SQLite database to identify suspicious activity correlations between targets within the same publication.
- `chembl_leakage.ipynb`: A Jupyter notebook for analyzing and visualizing the results found by `find_matches.py`.
- `muscarinic.csv`: Dataset from the muscarinic example mentioned in the paper.
- `results.csv`: The output generated by `find_matches.py`.

## Prerequisites

### Data Requirements
The script `find_matches.py` specifically requires **ChEMBL 36** in SQLite format. 

1. **Download:** [ChEMBL 36 SQLite (tar.gz)](https://ftp.ebi.ac.uk/pub/databases/chembl/ChEMBLdb/latest/chembl_36_sqlite.tar.gz)
2. **Setup:** Extract the `chembl_36.db` file and place it in this directory.

### Python Dependencies
The following libraries are required:
- `biopython` (for sequence alignment)
- `sqlite3` (standard library)
- `csv`, `math`, `collections` (standard library)

## How it Works

`find_matches.py` performs the following steps:
1. **Sequence Fetching:** Retrieves protein sequences for all 'SINGLE PROTEIN' targets.
2. **Activity Extraction:** Gathers pChEMBL values for compounds associated with specific documents (DOIs) and targets.
3. **Comparison:** For every document containing multiple targets, it identifies pairs of targets that share at least 10 compounds.
4. **Filtering:** It calculates the Pearson $R^2$ of the activity values and the global sequence similarity of the protein targets.
5. **Output:** Records instances where **$R^2 \ge 0.6$** despite **sequence similarity < 0.9**, suggesting that the activity profiles are unexpectedly similar.

## Usage

```bash
python find_matches.py
```

The results will be saved to `results.csv`, which can then be explored using the `chembl_leakage.ipynb` notebook.
